<p>Argo Workflows 4.0 is here!</p><figure><img alt="Argo octopuses loading boxes representing pods, steps and templates into data centre computers" src="https://cdn-images-1.medium.com/max/1024/0*vbCcas1di2_OTQRj.png" /></figure><p>This release brings a wave of usability, security and performance improvements that make running large‑scale pipelines easier than ever. Whether you’re orchestrating AI/ML workflows, scaling data pipelines, or driving CI/CD across many clusters, the new features in 4.0 give you the confidence to create and orchestrate workflows faster and safer.</p><h3>Release Statistics</h3><ul><li>🚀 24 new features</li><li>🐞 122 fixes</li><li>👥 73 contributors total (including 37 new contributors)</li></ul><h3>Argo Workflows 4.0 Release Highlights</h3><ul><li>Artifact drivers as plugins</li><li>CLI improvements — including creating semaphores</li><li>Full CRDs with validation</li><li>Deprecated singular items (mutex, semaphore, schedule ).</li></ul><h3>General</h3><ul><li>Artifact drivers as plugins — Write a gRPC server that acts as an artifact driver, ship it as a container image and plug it into Argo. Uploads and downloads can now target any repository you implement. A team effort from <a href="https://github.com/Joibel">Alan Clucas</a>,<a href="https://github.com/JPZ13"> J.P. Zivalich</a>, and<a href="https://github.com/elliotgunton"> Elliot Gunton</a> (<a href="https://github.com/argoproj/argo-workflows/issues/5862">#5862</a>)</li><li>CRD validation rules — CEL‑based validation is added to the full CRDs, catching many errors before they reach the cluster. Limits are enforced to stay within the Kubernetes CEL budget (e.g., max 200 templates per workflow, max 500 parameters). This huge feature has been delivered by <a href="https://github.com/Joibel">Alan Clucas</a> and <a href="https://github.com/MasonM">Mason Malone</a> (<a href="https://github.com/argoproj/argo-workflows/issues/13503">#13503</a>). <strong><em>For a detailed description of all the rules and constraints go to the Argo Workflows v4.0 release notes and documentation.</em></strong></li><li>Deprecate singular sync primitives — the singular mutex, semaphore and the CronWorkflowsschedule fields are removed; use the plural versions introduced in 3.6. By <a href="https://github.com/Joibel">Alan Clucas</a> (<a href="https://github.com/argoproj/argo-workflows/issues/14977">#14977</a>)</li><li>Name‑filter on archived workflows — A new nameFilter parameter (prefix | contains | exact) lets you search archived workflows the same way you do the active ones. Thank you <a href="https://github.com/arminfriedl">Armin Friedl</a> (<a href="https://github.com/argoproj/argo-workflows/issues/14069">#14069</a>)</li><li>Live update of total parallelism — Changing workflow-controller-configmap now takes effect immediately, no controller restart required. We owe this one to <a href="https://github.com/shuangkun">Shuangkun Tian</a> (<a href="https://github.com/argoproj/argo-workflows/issues/14689">#14689</a>) — much appreciated!</li><li>Custom CA certificates for OIDC SSO — Configure self‑signed or private‑CA certificates inline or via a file path, enabling secure SSO connections to non‑public providers. Thanks to <a href="https://github.com/bradfordwagner">Bradford Wagner</a> (<a href="https://github.com/argoproj/argo-workflows/issues/7198">#7198</a>)!</li><li>Write‑back informer disabled by default — Prevents unexpected controller behavior; 409 conflicts may increase slightly but overall stability improves. Shout outs go to <a href="https://github.com/eduardodbr">Eduardo Rodrigues</a> (<a href="https://github.com/argoproj/argo-workflows/issues/12352">#12352</a>)</li><li>Extended field‑selector support — metadata.name= and metadata.name!= are now accepted, giving you the same expressive power as native Kubernetes selectors. <a href="https://github.com/miltalex">Miltiadis Alexis</a> (<a href="https://github.com/argoproj/argo-workflows/issues/13468">#13468</a>)</li><li>Restart pods that fail before starting by <a href="https://github.com/Joibel">Alan Clucas</a> (<a href="https://github.com/argoproj/argo-workflows/issues/12572">#12572</a>) Automatically restart pods that fail before starting for reasons like node eviction. This is safe to do even for non-idempotent workloads. You need to configure this in your workflow controller configmap for it to take effect.</li><li>Context‑aware structured logging — Logs now carry workflow identifiers throughout the call stack, making troubleshooting far easier. Thank you <a href="https://github.com/isubasinghe">Isitha Subasinghe</a> (<a href="https://github.com/argoproj/argo-workflows/issues/11120">#11120</a>)</li><li>Feature documentation requirement — Every new feature must now have a .features/pending file describing its user‑visible impact. This streamlines release notes and blog generation. Added by <a href="https://github.com/Joibel">Alan Clucas</a> (<a href="https://github.com/argoproj/argo-workflows/issues/14155">#14155</a>)</li></ul><h3>UI improvements</h3><ul><li>CronWorkflow delete confirmation — The modal now warns that deleting a CronWorkflow will also delete the workflows it generated. Thank you <a href="https://github.com/miinsun">minsun yun</a> (<a href="https://github.com/argoproj/argo-workflows/issues/14679">#14679</a>)</li><li>Label query sync in WorkflowTemplates UI — URL query parameters (?label=key%3Dvalue) now stay in sync with the UI state, mirroring the behavior of the Workflows list. We owe it to <a href="https://github.com/puretension">Dohyeong Lee</a> (<a href="https://github.com/argoproj/argo-workflows/issues/14807">#14807</a>)</li><li>“Name Not Equals” filter — Exclude workflows by name directly from the list view, complementing the existing exact‑match filter. Done by <a href="https://github.com/miltalex">Miltiadis Alexis</a> (<a href="https://github.com/argoproj/argo-workflows/issues/13468">#13468</a>)</li><li>Custom link target handling — Links defined with target: _blank automatically open in a new tab; target: _self stays in the same tab. Thanks to <a href="https://github.com/shuangkun">Shuangkun Tian</a> (<a href="https://github.com/argoproj/argo-workflows/issues/13114">#13114</a>)</li><li>Faster pagination for archived workflows — Archive queries now use LIMIT‑based checks instead of full table scans, dramatically improving performance on large datasets. Good job <a href="https://github.com/shuangkun">Shuangkun Tian</a> (<a href="https://github.com/argoproj/argo-workflows/issues/13948">#13948</a>)</li></ul><h3>CLI improvements</h3><ul><li>Create semaphore configmaps from the CLI — argo semaphore create configmap … lets you provision a ConfigMap‑backed semaphore without manual YAML. Thank you <a href="https://github.com/djanjic">Darko Janjic</a> (<a href="https://github.com/argoproj/argo-workflows/issues/14671">#14671</a>)</li><li>Create database‑backed semaphores from the CLI — A matching argo semaphore create db … command adds support for external storage backends. <a href="https://github.com/djanjic">Darko Janjic</a> (<a href="https://github.com/argoproj/argo-workflows/issues/14783">#14783</a>) once more!</li><li>argo convert command — Automatically migrates existing Workflows, CronWorkflows and (Cluster)WorkflowTemplates to the new v4.0 schema: schedules move to the schedules field, singular sync primitives become plural, and the command performs lint‑style checks as it rewrites the objects. Done by <a href="https://github.com/Joibel">Alan Clucas</a> (<a href="https://github.com/argoproj/argo-workflows/issues/14977">#14977</a>)</li></ul><h3>Ready to try it out?</h3><p>Grab the latest container images, update your controller ConfigMap and start experimenting with the new name‑filter, artifact‑driver plugins and custom CA support. Share your feedback on the Argo community Slack or open an issue on GitHub — the next stable release is shaped by your experience.</p><p>Happy workflow building!</p><img alt="" height="1" src="https://medium.com/_/stat?event=post.clientViewed&amp;referrerSource=full_rss&amp;postId=70def0d672ef" width="1" /><hr /><p><a href="https://blog.argoproj.io/argo-workflows-4-0-rc2-70def0d672ef">Argo-Workflows 4.0</a> was originally published in <a href="https://blog.argoproj.io">Argo Project</a> on Medium, where people are continuing the conversation by highlighting and responding to this story.</p>
